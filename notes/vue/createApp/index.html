# 全局API

在Vue2.0中，我们会使用new Vue()来创建新的Vue实例。但是这种方法存在一定的弊端：

例如现在有一个很大的Vue应用，小红需要Vue_A实例对象，并且这个对象拥有全局组件EnglishBook，小蓝需要一个Vue_B组件，但是小蓝不喜欢英语，他不需要EnglishBook，要求这两个Vue实例的功能完全独立，相互隔离。

但是在Vue2.0中通过new Vue() 连续实例化两个实例，再注册一个全局组件，这种方法是行不通的。因为在Vue2.0中，所有的Vue实例是共享一个构造函数的（包括全局指令/全局组件等），无法做到完全隔离

Vue3.0 中新增了createApp方法，该方法返回一个应用实例，createApp方法可以返回一个提供应用上下文的应用实例，应用实例挂载的整个组件树共享同一个上下文。

iimport { createApp } from 'vue'
import App from './App.vue'
 
createApp(App).mount('#app')

在Vue3.0中，可以解决上述的问题：

<template>
    <div id="little-red"></div>
    <div id="little-blue"></div>
</template>

<script>
    const vueA = Vue.createApp({
        template: `
            <div>
                vueA
                <english-book></english-book>
            </div>
        `
    })
    vueA.component('english-book', {
        template: `<p>English Book</p>`
    })
    vueA.mount('#little-red')

    const vueB = Vue.createApp({
        template: `<div>vueB</div>`
    })

    vueA.mount('#little-blue')
</script>

这样，组件vueA和vueB是完全隔离的环境，english-book组件在vueA实例控制的视图中是完全全局可用的，而在vueB控制的视图区域是不可用的，淡然我们也可以把english-book组件分别在两个实例上注册。我们可以把案例中的vueA 和 vueB想象成俩个完全不同的应用实例，它们各自为营，互不影响，完全隔离，同样也可以自由的选择要共享组件。
